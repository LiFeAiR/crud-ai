## Что было реализовано:

1. **Добавлена поддержка JWT токенов**:
    - Создана структура Claims для JWT токена с user_id, email, name и стандартными claims
    - Реализована генерация и валидация JWT токенов с использованием HS256 алгоритма

2. **Добавлен новый RPC метод Login**:
    - В proto файл добавлены сообщения LoginRequest и LoginResponse
    - Добавлен метод Login в сервис CrudService с REST API endpoint /api/login

3. **Реализована бизнес-логика авторизации**:
    - Метод Login в базовом обработчике проверяет email и пароль
    - При успешной проверке генерируется JWT токен
    - Возвращается токен и информация о пользователе

4. **Обновлены компоненты системы**:
    - Интерфейс UserRepository дополнен методом GetUserByEmail
    - Реализация GetUserByEmail в репозитории пользователей
    - Добавлены тесты для метода авторизации

5. **Тестирование**:
    - Созданы тесты для проверки успешной авторизации
    - Тесты на ошибки при неверных учетных данных
    - Тесты на ошибки в работе с репозиторием

## Как использовать:

1. Отправьте POST запрос на `/api/login` с параметрами:
   ```json
   {
     "email": "user@example.com",
     "password": "user_password"
   }
   ```

2. В ответ получите:
   ```json
   {
     "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
     "user": {
       "id": 1,
       "name": "John Doe",
       "email": "user@example.com"
     }
   }
   ```

3. Для последующих запросов используйте токен в заголовке Authorization:
   ```
   Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
   ```
